How Cloud Based License Management System Works

How it works

  • First register to get unique user api key to your email
  • Login happens with only the unique user api key
  • Create new license by pressing + button and change the subscription type for your client.
  • The new license record shall have unique serial no.
  • When your client enters the unique serial number your software shall send serial number and your user api key to activate url in post body
  • Activate url can be seen in the test bench when you login in.
  • This will activate the license and send your software back product_key and expiry date.
  • Your software shall store this product_key in it private memory
  • To de-activate the license, your software shall send serial number and product key along with your user api key to de-activate url in post body
  • You will receive status feedback for de-activation
  • Your client can deactivate to move the software to new device and then re-activate
  • Internet connection is required only at the time of activating and de-activating your software
  • Product key sent to your software at the time of activation is every time unique and require at the time of de-activation
  • Hardware code can be stored and checked by your software to ensure the software is not running on wrong machine. This activity is by your end
  • Value proposition of this cloud application is to provide you ability to extent yor software license to multiple machines with only definite number of user licenses.
  • Also this cloud application can provide you with serial number and product key for your license
  • Try it yourself in the test bench after login in
